## 项目概览

Some-tools 用于收纳体量较小、目标明确的 Windows 工具和桌面实验。每个工具从一个可重复描述的问题出发，尽量保持依赖简单、启动直接、便于修改。

::github{repo="kotorin-Y/Some-tools..."}

## 这个仓库体现什么

- 能够快速识别重复劳动并做出最小可用工具。
- 使用 Python 处理文件、桌面交互与自动化场景。
- 在小项目中同样关注错误提示、使用说明和可复现性。

[打开 GitHub 仓库](https://github.com/kotorin-Y/Some-tools...)

## 评论抓取工具

这是一个更偏运营资料整理的小工具。它的目标很直接：把网页评论样本抓下来，整理成后续可分析的文本材料，方便做用户反馈归类、评论情绪观察和内容复盘。

我把它单独放在 `some-tools` 里，是因为它不属于完整产品，也不是一篇报告。它更像一个小型工作流组件：当我需要研究社区讨论、评论区反馈或用户表达时，可以先用工具把样本收集下来，再进入人工判断和归类。
