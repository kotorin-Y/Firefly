# kotorin 个人博客作品集与资源中心改版规格

日期：2026-06-25  
状态：已完成方案确认，等待书面审查  
目标仓库：personal-blog / Firefly

## 1. 项目目标

将现有个人博客改造为面向游戏运营、产品运营、产品经理岗位的求职作品站。网站需要兼顾三类目标：

1. 招聘者可以在较短时间内理解 kotorin 的职业方向、能力结构和可量化经历。
2. 每个项目以完整案例而非单张效果图呈现，展示从业务问题、策略设计到产品落地和质量验证的全过程。
3. 所有可公开交付物均通过博客站内链接下载，包含程序、源码、PRD、交互素材和专项运营方案。

站点公开身份统一使用昵称 `kotorin`，不展示真实姓名、私人邮箱、所在地或未获授权的个人信息。公开联系入口仅使用现有 GitHub 等公开渠道。

## 2. 目标受众与阅读任务

### 2.1 主要受众

- 游戏公司招聘负责人及业务面试官。
- 游戏运营、产品运营、内容运营、社区运营岗位负责人。
- 产品经理、项目经理及相关协作岗位面试官。
- 对项目源码或交付物感兴趣的开发者和 ACGN 同好。

### 2.2 核心阅读任务

- 判断候选人是否具备用户洞察、活动策划、内容组织和项目推进能力。
- 判断候选人能否将运营需求转化为可操作的产品或页面。
- 核验数据意识、复盘方法、跨团队协作和工程理解能力。
- 下载并检查 PRD、设计素材、源码或 Windows 程序。

## 3. 内容策略与语气

文案采用“专业产品与运营表达 + ACGN 爱好者热情”的组合：

- 使用用户、场景、问题、目标、指标、策略、方案、验收、复盘等专业术语。
- 不使用空泛的“热爱互联网”“执行力强”等自我评价，能力必须由经历和项目证据支撑。
- 允许适度使用 🎮、✨、🧩、📊、🚀 等 Emoji，主要用于导航、标签和段落提示，不替代正文信息。
- 对非官方概念项目明确标注学习与求职展示属性，避免造成官方合作或授权误解。
- 不虚构 Figma、Pencil 或其他不存在的源文件。没有真实设计源文件时，以截图、流程图、设计令牌、素材清单和交互说明组成“交互设计素材包”。

## 4. 信息架构

### 4.1 顶层导航

- 首页
- 个人作品集
- 关于
- 文章归档或现有博客辅助页面
- GitHub

“个人作品集”是四个项目的唯一集中入口。首页不再重复展示项目卡片或项目列表，只保留个人定位、内容文章和一个进入作品集的明确入口。

### 4.2 个人作品集叙事

作品集采用“策展叙事型”结构：

1. 求职定位：游戏运营 / 产品运营 / 产品经理。
2. 能力地图：用户洞察、内容与活动运营、产品设计、数据分析、工程落地、质量验证。
3. 项目旅程：发现问题 → 制定策略 → 设计体验 → 搭建产品 → 验证交付。
4. 四个项目案例。
5. 可下载交付物说明和使用边界。

四个项目：

- OpsDeputy：本地优先的运营 AI Agent。
- Windows Motion Studio：Windows 桌面动态与交互创作工具。
- Fancy!：ACGN 综合资讯、搜索与共创桌面产品。
- Stellar Blade: BLOOD RAIN：游戏全球宣发概念网页及用户与社区运营方案。

## 5. 项目详情页统一模板

每个项目页面均包含：

1. 项目头图、名称、项目类型、状态和职责标签。
2. 业务背景与用户问题。
3. 产品或运营目标。
4. 我的职责与职责边界。
5. 用户画像或核心场景。
6. 关键策略和方案。
7. 功能模块或页面结构。
8. 技术栈和工程结构。
9. 3 至 6 张真实项目截图或素材图。
10. 结果、验证过程和复盘。
11. 下载中心。
12. 版权、授权和演示性质说明。

页面内容强调业务判断，不将技术栈作为唯一价值。开发实现需要解释其如何服务信息效率、内容质量、稳定性或交付复现。

## 7. 下载中心规则

所有下载资源位于博客 `public/downloads/portfolio/<project>/`。

文件命名使用英文小写或清晰版本号，避免空格和本地路径：

- `<project>-setup-vX.Y.Z-x64.exe`
- `<project>-portable-vX.Y.Z-x64.exe`
- `<project>-prd-zh-cn.pdf`
- `<project>-interaction-assets-vX.zip`
- `<project>-source-vX.zip`
- `stellar-blade-blood-rain-community-operations-plan-zh-cn.pdf`

每个下载项展示：

- 文件名称。
- 类型。
- 版本。
- 文件体积。
- 适用平台。
- 内容用途。
- 运行或版权提示。

源码包必须排除：

- `.git`
- `node_modules`
- 构建缓存
- 临时文件
- 用户数据
- 日志和 QA 运行目录
- 本地绝对路径
- 令牌、密钥和 `.env`

博客使用 GitHub Pages 构建与发布。所有资源随仓库提交。发布前检查 GitHub 单文件限制、仓库体积和 Pages 构建产物完整性。

## 8. 首页设计

首页不展示四个项目卡片，避免与作品集重复。

首页包含：

- 使用新背景图的沉浸式 Banner。
- 一句话定位：面向游戏运营、产品运营和产品经理岗位。
- 简短介绍：将用户洞察、运营策略和产品搭建转化为可验证交付。
- “进入个人作品集”主按钮。
- 博客文章流和现有辅助内容。

背景图使用用户提供的 `83647245_p0.png`，复制到博客公开资源目录后引用。禁止在代码或页面中出现本地绝对路径。

视觉方向：

- 海蓝作为主色。
- 雾白作为正文卡片底色。
- 樱粉作为 CTA 和情绪强调色。
- 正文区域使用足够不透明度或磨砂层确保可读性。
- 不让背景中的高亮白色水母干扰正文标题。

## 10. 组件与实现边界

优先复用 Firefly 现有文章系统、Markdown/MDX 渲染、图片组件、下载链接和导航结构。

新增或调整：

- 作品集总览 MDX 页面。
- 四个项目详情 MDX 页面。
- 可复用下载资源卡组件。
- 可复用项目指标、职责、技术栈和截图画廊组件。
- 关于页内容。
- 背景配置与站点文案。
- 首页内容筛选或结构调整，确保作品项目不重复展示。

避免为作品集创建独立 SPA 或第二套餐式系统。内容数据尽量从 MDX frontmatter 或项目配置对象读取，便于后续维护。

## 11. 错误处理与降级

- 下载文件缺失时构建必须失败或由校验脚本报告，不能发布死链。
- 图片缺失时使用项目封面或统一回退图。
- 大文件必须在构建和部署前检查大小。
- EXE 下载项必须提示仅支持 Windows，并说明未配置商业代码签名时可能触发 SmartScreen。
- 外部 GitHub 链接仅作为补充，站内下载链接为主要入口。
- PDF 或 ZIP 无法生成时，不使用空白占位文件；先报告缺失并停止对应项目发布。

## 12. 安全、版权与隐私

- 不上传密钥、令牌、浏览器缓存、用户数据或本地绝对路径。
- 对游戏名称、角色、媒体图像和第三方品牌标注权利归属。
- 对概念项目明确非官方性质。
- 源码包和设计素材包在打包前进行路径、密钥和大文件扫描。
- 用户提供的背景图作为博客视觉资产使用；博客内不声明原创或版权归属，除非用户后续提供授权信息。

## 13. 验收标准

### 13.1 内容

- 作品集包含四个项目，且首页不重复展示项目列表。
- 每个项目均包含背景、目标、职责、方案、技术栈、截图、复盘和下载区。
- 关于页面包含完整求职定位、两段工作经历、量化结果和技能映射。
- 文案同时具备专业性与 ACGN 爱好者人格，但不过度使用 Emoji。

### 13.2 下载

- 所有页面下载按钮可实际下载文件。
- OpsDeputy、Windows Motion Studio、Fancy 提供 EXE、PRD、设计素材包和源码包。
- Stellar Blade 项目提供源码包、页面素材包和用户与社区运营方案 PDF。
- 文件名、版本、体积、平台和用途展示准确。

### 13.3 视觉和交互

- 新背景图在桌面端、平板和手机端裁切合理。
- 作品集总览与项目详情信息层级清晰。
- 下载卡、截图画廊、按钮和导航具备悬停、焦点与移动端状态。
- 390 px、768 px、1280 px 和 1440 px 宽度无横向溢出。
- 正文对比度和背景磨砂不影响长文阅读。

### 13.4 工程

- `pnpm check` 或等效 Astro 检查通过。
- 生产构建通过。
- 下载资源校验脚本通过。
- 页面控制台无错误，站内链接和下载链接无死链。
- Git diff 中不存在本地路径、密钥、用户数据或不应公开的运行记录。
- GitHub Pages 部署成功，并对线上作品集和至少一个大文件下载进行验证。

## 14. 非目标

- 不建立账号、支付、在线编辑器或后端下载统计系统。
- 不虚构不存在的 Figma、Pencil 或 Adobe 源文件。
- 不在首页重复维护第二份项目数据。
- 不重新设计 Firefly 的全部主题和博客基础架构。
- 不将求职作品站包装成真实商业上线产品或官方合作项目。

1. 清点和筛选每个项目可公开资源。

4. 引入背景图并更新站点文案与关于页。
5. 实现作品集总览、四个项目详情页及下载组件。
6. 更新首页去重逻辑和导航。
7. 执行构建、响应式、下载、安全和部署验证。
